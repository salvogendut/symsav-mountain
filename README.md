# symsav-mountain

A 3D isometric terrain screensaver for [SymbOS](https://www.symbos.org/) on the Amstrad CPC and MSX.

> **Alpha version — use at your own risk.** This software is in an early alpha state and may cause harm to your system. If you choose to try it, you do so entirely at your own risk.

> **CPC: Requires Mode 1** — on the Amstrad CPC this screensaver only works in 320×200 Mode 1 (4 colours). Running it in any other screen mode will produce incorrect output.

Inspired by Pascal Pensa's `mountain` module from the [xlockmore](https://www.tux.org/xlockmore/) / xscreensaver suite (1995).

---

## Building

```bash
./build.sh
```

Requires the SCC compiler (set `SCC=` env var if not at `../scc/bin/cc`) and Python 3.

Build steps:

1. SCC compiles `mountain.c` → `mountain.sav`
2. `add_preview.py` patches the preview thumbnail into the binary at file offset 256

Output: `mountain.sav`

---

## Installing

1. Copy `mountain.sav` into your `C:\SYMBOS\` directory.
2. Open **Display Properties** and go to the **Screen Saver** tab.
3. Click **Browse** and select `mountain.sav`.
4. Click **Setup** to configure the effect:
   - **Speed**: Slow / Normal / Fast — controls how many grid cells are drawn per tick
   - **Peaks**: Few / Normal / Many — number of random height peaks seeded into the terrain

---

## Effect

- A 20×20 terrain height map is generated by scattering random peaks and smoothing them twice with a neighbourhood spread/average pass, then adding a small amount of micro-noise.
- The terrain is projected onto the full screen using the same isometric formula as the original `mountain.c` and drawn as a wireframe grid, one cell at a time from back to front.
- Cell edges are coloured by the average height of their four corners:

**CPC (Mode 1, 320×200, 4 colours):**

| Height range | Ink | Appearance |
|---|---|---|
| Low (valley floors) | 2 (dim) | dark lines |
| Mid (hillsides) | 3 (bright) | bright lines |
| High (peaks) | 0 (white) | white / snow lines |

**MSX (Screen 7, 512×212, 16 colours):**

| Height range | Colour | Appearance |
|---|---|---|
| Low (valley floors) | COLOR_GREEN | dark green lines |
| Mid (hillsides) | COLOR_LGREEN | light green lines |
| High (peaks) | COLOR_WHITE | white / snow lines |

- After the full grid is drawn, the terrain is held on screen for a few seconds, then cleared and a new random landscape is generated.

## Screensaver protocol

Standard SymbOS screensaver messages:

| Message | Action |
|---------|--------|
| `MSC_SAV_INIT` (1) | Load saved config from manager |
| `MSC_SAV_START` (2) | Start fullscreen animation |
| `MSC_SAV_CONFIG` (3) | Open config dialog |
| `MSR_SAV_CONFIG` (4) | Send updated config back |

Config is 6 bytes: magic `"MNTN"` + speed byte + peaks byte.

## Animation

Fullscreen rendering follows the same approach as [symsav-xroach](https://github.com/salvogendut/symsav-xroach) and [symsav-xmatrix](https://github.com/salvogendut/symsav-xmatrix):

1. Open a fullscreen `WIN_NOTTASKBAR | WIN_NOTMOVEABLE` window
2. `DSK_SRV_DSKSTP` to freeze the desktop
3. Clear the screen
4. Per tick: draw N terrain cells using a Bresenham line algorithm, one pixel at a time
5. Exit on any key or mouse movement: resume desktop, close window, `Screen_Redraw()`

Platform is detected at runtime via `Sys_Type()`.

### CPC rendering

- Screen clear: `Bank_Copy` all 8 Mode-1 scan planes to `0xF0` (ink 1 = black)
- Per-pixel write: read-modify-write via `Bank_Copy` into VRAM bank 0
- VRAM address: `0xC000 + (y/8)*80 + (y%8)*0x800 + x/4`
- Mode-1 byte encoding (4 pixels per byte):
  ```
  bit7=p0_lo  bit6=p1_lo  bit5=p2_lo  bit4=p3_lo
  bit3=p0_hi  bit2=p1_hi  bit1=p2_hi  bit0=p3_hi
  ```
- Lower half (y ≥ 96) uses a write-through shadow buffer (`lbuf`) to survive SymbOS kernel VRAM writes during `Idle()` — see [VRAM corruption and the shadow buffer](#vram-corruption-and-the-shadow-buffer) above.

### MSX rendering

- Screen clear: `vdp_fill(0, 0x11, 54272)` — fills all 256×212 bytes of Screen 7 VRAM
- Per-pixel write: `vdp_fill(y*256 + x/2, color_byte, 1)` — writes both nibbles of a byte as the same colour (2-pixel-wide lines), no VDP read required
- VDP ports: `0x98` (data), `0x99` (address / register write); R#14 sets VRAM bank bits [16:14]; the 17-bit address counter auto-increments across 16 KB bank boundaries
- No shadow buffer needed — the MSX kernel does not corrupt VRAM during `Idle()`

### Isometric projection

Scales to the active screen dimensions at runtime (`screen_w` × `screen_h`):

```
x2 = gx * (2 * screen_w) / (3 * 20)
y2 = gy * (2 * screen_h) / (3 * 20)
screen_x = (x2 - y2/2) + screen_w/4
screen_y =  y2 - height + y_offset
```

| Platform | screen_w | screen_h | y_offset |
|---|---|---|---|
| CPC | 320 | 200 | 70 |
| MSX | 512 | 212 | 78 |
